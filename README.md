📡 rf_baseline_observer
Data Center Interference Monitoring System (OpenClaw + RTL-SDR)
This project establishes a high-resolution RF (Radio Frequency) noise floor baseline on Debian to quantify electromagnetic interference (EMI) generated by the construction and operation of a nearby data center.

🏗 Project Architecture
Sensor: RTL-SDR (Blog V4 or similar)

Host: Debian (Headless Console)

Data Logger: rtl_power (Wideband signal integration)

Processor: check_floor.py (Python/Pandas analysis)

Analyst: OpenClaw (Autonomous AI Agent)

🚀 Installation & Setup
1. System Preparation
Run these commands in your Debian console to prepare the hardware:

# Install dependencies
sudo apt update && sudo apt install rtl-sdr python3-pandas python3-numpy tmux git -y

# Blacklist TV drivers to free up the SDR
echo "blacklist dvb_usb_rtl28xxu" | sudo tee /etc/modprobe.d/blacklist-rtl.conf
sudo modprobe -r dvb_usb_rtl28xxu

2. Analysis Script (check_floor.py)
Create this script in your home directory to process the raw CSV data into human-readable metrics.

Python
import pandas as pd
import numpy as np
import sys
import os

def analyze_rf_csv(file_path):
    if not os.path.exists(file_path):
        print(f"Error: {file_path} not found.")
        return

    try:
        df = pd.read_csv(file_path, header=None)
        db_data = df.iloc[:, 6:].values.flatten()
        db_data = db_data[~np.isnan(db_data)]
        
        avg_floor = np.mean(db_data)
        peak_floor = np.max(db_data)

        print(f"--- RF NOISE REPORT: {os.path.basename(file_path)} ---")
        print(f"Average Noise Floor: {avg_floor:.2f} dBm")
        print(f"Peak Interference:   {peak_floor:.2f} dBm")
        
        if avg_floor > -60:
            print("STATUS: CRITICAL - High environmental noise detected.")
        elif avg_floor > -75:
            print("STATUS: WARNING - Elevated noise floor.")
        else:
            print("STATUS: NOMINAL - Baseline levels maintained.")
    except Exception as e:
        print(f"Error: {e}")

if __name__ == "__main__":
    analyze_rf_csv(sys.argv[1])
📊 Operation Workflow
Phase 1: Background Logging
Start a long-term sweep in a tmux session. This ensures the scan continues even if you log out.

tmux new -s baseline
rtl_power -f 24M:1.7G:1M -i 1h -e 7d ~/pre_dc_baseline.csv

Press Ctrl+B then D to detach.

Phase 2: Automated Summaries
Add a cron job to generate a timestamped history log for OpenClaw to review.

(crontab -l ; echo "0 * * * * /usr/bin/python3 ~/check_floor.py ~/pre_dc_baseline.csv >> ~/rf_history.log 2>&1") | crontab -

Phase 3: OpenClaw Integration
Configure your OpenClaw agent with the following directive:

"You are an RF Analyst. Access the log at ~/rf_history.log. When asked for an update, analyze the trend. Report if the Average Noise Floor is trending upward, which indicates the Data Center is increasing its power load."

🛠 Troubleshooting
SDR not found: Run rtl_test -t. If it fails, check USB connections.

Permission Denied: Ensure your user is in the plugdev group.

Inaccurate Readings: Ensure the antenna is placed away from local electronics (PCs, Routers) to capture external DC noise effectively.
